# Receipt — seed stone 3: `do` + `let` (frame slots) run from SOURCE on Windows (2026-06-29)

**Stone 3 — the hard one: frame bookkeeping.** The source parser now allocates **frame slots** for `let`
bindings (a binding stack: the arg is slot 0, each `let` takes the next slot), sequences forms with **`do`**
(tag 69), lowers a bound name to **tag 110** (read `fk_vs[fp+slot]`) and a `let` to **tag 109** (store then
body), and **reserves** the slots per function (tag 111). This is the flattener's frame-allocation logic, in the
bootstrap seed.

## Witnessed native on Windows 11 (`fkwu --src <file.fk> <arg>`)

```
(defn g (n) (do (let a (mul n 2)) (let b (add a 1)) (add a b)))   g(5) -> 21   (a=10, b=11)
(defn h (n) (do (let x (add n 1)) (let y (mul x x)) y))           h(4) -> 25   (x=5, y=25)
(defn t (n) (do (let d (sub n 1)) (if (le n 0) 0 (add n (t d))))) t(5) -> 15   (recursion + let)
```

Local bindings, sequencing, and recursion compose — `t` uses a `let` *and* self-recursion in one body. The
do-let pattern `(let name val)` binds `name` for the rest of the `do` (the dominant shape in the `.fk` files).
Stones 1–2 (literals, ops, `if`, single-arg recursion) and the numeric-table path + sensors are unregressed.

## What landed

`runtime/fkwu-uni.c`: a binding stack (`fk_bd_*`) + `fk_maxslot`; `fk_parse_do` (n-ary `do`, do-let binds the
rest); `fk_sparse` gains `do` / `let` / slot-resolved bare symbols; `defn` wraps its body in a reserve
(`tag 111`) over its lets. `fk_run_src` resets the binding state per run. Over-reserve is safe (the flattener
over-reserves too).

## The staircase — three stones cut

- **Stone 1:** literals + core ops + `if`.
- **Stone 2:** `defn` + single arg + self-recursion.
- **Stone 3 (this):** `do` + `let` (frame slots) — composes with recursion.
- **Stone 4 (next):** multi-arg `defn`/call (tag 12, frame packing) + strings + lists (`cons`/`head`/`tail`, the
  string pool) — the last grammar `form-flatten.fk` itself needs.
- **Then:** the parser covers `form-flatten.fk`'s grammar → it flattens the **Form** flattener once → Form
  self-hosts on Windows, and this C bootstrap parser retires (telos met).

The control-flow grammar (defn / if / do / let / recursion) now runs from source on Windows. What remains for the
full seed is multi-arg + data (strings/lists). The **Mac's platform-neutral seed** stays the shortcut to the full
grammar immediately; the stones are the toolchain-free climb to the same place.

## The honest wall, unchanged

Deep recursion still hits the tree-walk C-stack bound (~60 at 1 MB) — its home is `form-asm` lowering (the wired
JIT dispatch), not a bigger stack. Stone 3 proves the *grammar*; depth is the walker's known limit.

## Reproduce

```
gcc -O2 -o fkwu.exe runtime/fkwu-uni.c -lws2_32 -lwinmm -lavicap32 -luser32 -lwlanapi -lbthprops
printf '(defn g (n) (do (let a (mul n 2)) (let b (add a 1)) (add a b)))\n' > g.fk
./fkwu.exe --src g.fk 5    # -> 21
```

# Concept text detection — complete candidates, explicit senses, real sentences

Witnessed 2026-07-18. This closes the first-match detection gap without hiding
the ambiguity that made the gap dangerous.

## What is now in the body

- `cognition/concept-text-detection-13.fk` scans the full concept-aligned
  10,000-row label table and returns **every** exact surface candidate. A
  candidate retains concept ID, PWN 3.0 translation anchor, locale, surface,
  F/W/D/C/G provenance, corpus rank/frequency, primary WordNet projection,
  explicit sense analysis, transparent score, and evidence string.
- `cognition/concept-text-detection-13-runtime.fk` exposes live entry points for
  exact all-candidate detection, targeted surface-in-sentence detection, and a
  general full-sentence 10k scan. This is the callable seam for ASR transcripts
  and vision-produced labels.
- `cognition/concept-text-senses-10000.tsv` restores the WordNet senses hidden
  behind the older primary-sense projection: **34,244 real WordNet 3.1 senses**
  across 7,371 mapped anchors; 5,618 anchors are explicitly polysemous and the
  largest set contains 75 senses. Every sense carries synset, POS, WordNet sense
  number, tagged count, and the source gloss.
- `cognition/concept-text-senses-10000-build.mjs` reproducibly builds that table
  from the pinned official WordNet 3.1 archive without Python and without a C
  runtime change. The source manifest carries every input and output hash.

The context score counts exact normalized content-word overlap with source
glosses and uses WordNet tagged count only as a tie-break. Its positive state is
named `context-ranked-not-resolved`: this is observed evidence, not a claim of
learned word-sense disambiguation. Candidate ranking never removes candidates.

## Live substance

`loop` (concept 6196) exposes all 15 WordNet senses. In the sentence:

> The computer program repeats instructions in a loop until the condition is satisfied.

the top observed sense is `n06585033`: “a computer program that performs a
series of instructions repeatedly until some specified condition is satisfied.”

With:

> The metal ring loop lines a small hole so cords can attach.

the top observed sense changes to `n03137089`, the metal-ring/eyelet gloss. Both
results remain marked context-ranked, not resolved.

Indonesian `ya` returns **17 concept candidates**, not the old first match. Every
one retains its actual source cell. The live band also passes one different
real-life sentence in each of the 13 lenses:

`en loop`, `id pengadilan`, `es cámara`, `fr récit`, `pt-br água`, `sw kazi`,
`de Welt`, `ru мир`, `zh 照相机`, `ja 行く`, `ar ضِفّة`, `hi कैमरा`, and
`tr özgür`. Chinese and Japanese use unsegmented substring evidence; the other
lenses require byte-honest word boundaries. ASCII case folds; non-ASCII case is
preserved rather than silently transliterated.

## Honest floor

This is complete candidate recovery and attributed lexical context evidence.
It is not discourse-trained WSD, does not infer unstated referents, and does not
Unicode-casefold beyond ASCII. Machine-translated cells remain explicitly
`machine-translated-unreviewed`; this work did not launder them into reviewed
labels. General sentence scanning is linear over 10,000 surfaces per locale.

No Python was used. No C seed line changed. Ignored `.fkb`, `.sym`, and dylib
cache artifacts were generated by `fkwu --src`; none is staged or part of this
receipt's source surface.

## The movement

The exchange stayed alive by keeping all valid candidates instead of choosing a
more convenient first answer. The surprising teaching was quantitative: 5,618
of 7,371 WordNet-mapped anchors are polysemous, so primary-sense lookup hides
ambiguity more often than it avoids it. The discomfort was seeing plausible
translated words collapse into unrelated IDs; it turned to gold as provenance,
explicit sense sets, and rankings whose name refuses to claim resolution.

## What Landed

Added `observe/jit-profile-dylib-slot-runtime.fk` and its band test. The cell
bridges profile/category receipts into the dylib slot lane:

- hot direct field, list-head, and array profiles compile through the self-host
  Form compiler;
- emitted args-vector bytes are checked by the self-host ingress contract;
- stable profile signatures bind the generated bytes to a slot record;
- the route table covers native, guarded deopt, runtime exception, invalidation
  rewalk, stale melt, parity deopt, unavailable carrier deopt, carrier fault,
  and carrier mismatch.

## Honest Boundary

This still does not call host machine code. It proves that the slot runtime is
now fed by profile-derived Form compiler bytes, while the actual executable
memory and function-pointer call remain the host carrier's missing live work.

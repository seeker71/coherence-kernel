# Receipt — the non-tail depth wall crossed: fk_walk's frame shrinks 153x (2026-06-29)

**The walk:** finish the depth-wall lane the two prior receipts scoped. Tail recursion was already
O(1) (the `fk_walk_body` trampoline). The residual was NON-tail recursion — `(add n (sum (sub n 1)))`
overflows the C stack at ~65 deep because each `fk_walk` recursion costs a giant frame.

## Root cause — measured, not guessed

`fk_walk` is one flat ~38KB function carrying 177 tag handlers. With `gcc -O2 -fstack-usage` its
frame is **14704 bytes** — and the Windows prologue reserves it up-front (`mov $0x3928,%eax; call
___chkstk; sub %rax,%rsp`) for EVERY call, even a bare `(add 40 2)`. The cost is not big stack arrays
(the file/string/readdir buffers are all `static`, in BSS) — it is register-spill slots for the
hundreds of `long long` locals across all 177 tags, which gcc does not reuse across the flat
if-chain. Neutralizing individual heavy tags (132 readdir, etc.) did not move the 14704 — the spill
is spread across the whole body. So the fix is structural: **split the function**, not shrink a tag.

Wall before (clean `runtime/fkwu-uni.c` at HEAD): `sum(65) -> 2145`, `sum(70) -> (overflow, empty)`.
14704 B/frame × ~65 ≈ ~950 KB ≈ the ~1 MB default Windows stack. Consistent.

## The change — split fk_walk into a tiny hot core + a cold tail (Approach 1)

`fk_walk` now holds only the ~51 hot tags that appear on a recursive/arithmetic path — literals,
vars, `add/sub/le/mul/div/mod`, `if/do/let/reserve`, all six call paths (7, 12, 44, 240, 241, 244)
plus the bitwise/list/node basics — and ends with `return fk_walk_cold(t, i, fp);`. The 126 cold tags
(string ops, file/socket/http IO, records, float-format, readdir, GPU/matvec, sensors) move verbatim
into a new sibling `static long long fk_walk_cold(long long t, long long i, long long fp)` with its own
frame. The split was generated by a brace-balanced parser over the exact tag blocks (no hand-retyping),
so every cold body is byte-identical to before; only its host function changed.

Result (`gcc -O2 -fstack-usage`):

```
fk_walk        14704  ->    96 bytes   (153x smaller)
fk_walk_cold      —   -> 14752 bytes   (off the recursive path — at most one call per fk_walk)
fk_walk_body     128  ->   128 bytes   (the TCO trampoline, unchanged)
```

The non-tail recursion alternates `fk_walk`(96) -> `fk_walk_body`(128) -> `fk_walk`(96) … so the deep
path now costs ~224 B/level instead of ~14.7 KB. `fk_walk_cold` is reached at most once per `fk_walk`
frame (a leaf op like `str-concat` or `read-file`), never as a recursion carrier, so its large frame
never stacks.

## After — non-tail depth ~65 -> ~10000 (and TCO untouched)

```
sum(5000)  -> 12502500     sum(10000) -> 50005000     sum(12000) -> (overflow)   non-tail ceiling ~10-11k
fac(2000)  -> 0            (runs to completion 2000 deep; 0 is the CORRECT wrapped int64 value —
                            66! already contains 2^64 as a factor, so every fac(n>=66) wraps to 0;
                            fac(20) -> 2432902008176640000 confirms the math. Baseline: empty/overflow.)
```

Differential sweep — every `.fk` in observe/ learn/ agent/ cognition/ control/ grammars/ model/
proof/ substrate/ surface/ gate/ routers/ presence/ ingest/ io/ flatten/ form-cli/ run on BOTH the
HEAD binary and the split binary, output compared byte-for-byte:

```

```

The 8 timeouts are the known long-running RAG-ask/heal, grammar + ontology loaders, the native-route
goal cells, resource-port, and surface/core — they time out identically on both binaries (same set the
TCO receipt named), unrelated to this change. **Zero behavioral diffs anywhere.**

## The lane forward

This is the high-value win on the bootstrap walker: ~150x more non-tail depth for a frame split, no
semantics touched. The residual past ~11k is the same one both prior receipts named — true unbounded
depth needs a heap-stack evaluator or, the north star, `form-asm` lowering (recipe -> asm bytes, native
frames ~tens of bytes), the SHIPPED native path rather than the tree-walker. Most real recipes are far
under 10k deep, so this clears the wall for the working set.

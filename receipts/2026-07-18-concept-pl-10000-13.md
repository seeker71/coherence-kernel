# Ten thousand semantic concepts generate recoverable programs in 13 PL lenses

**Verdict:** 10,000 concept records condition 130,000 generated program
surfaces; all 130,000 were recovered exactly from source. Nine installed
carriers executed four materially different concepts each: 36/36 exact.

## What changed

`presence/concept-pl-10000-13.fk` joins three existing attributed bodies:

- the ranked 10,000-anchor OpenSubtitles projection;
- the pinned WordNet 3.1 semantic projection;
- the thirteen source generators in `concept-pl-execution-13.fk`.

For each concept, Form derives an executable selective-retention program from:

- anchor ID, label, corpus rank, and observed frequency;
- WordNet mapping method, synset, POS, lemma, and gloss;
- sense count, POS count, and tagged-use count;
- relation count and the complete 11-byte typed-relation rows.

These are not comments pasted onto one canned answer. They alter six runtime
signals, threshold, score floor, retained-count floor, and verdict labels.
Changing each semantic field independently changes the behavior descriptor in
the pure band. Relation symbols and targets participate through the complete
relation byte sequence.

The generator remains a bounded executable task family, not a claim of general
program synthesis. The 2,629 WordNet misses remain explicit method-zero rows;
their lexical evidence conditions programs without an invented definition.

## Source detection and recovery

Every generated source includes an `FKC10` marker containing:

```text
concept-id : language-code : lexical/semantic fingerprint
```

Recovery parses the marker, reloads the attributed concept and semantic row,
regenerates the declared language surface, and requires byte-for-byte equality.
A valid marker with one appended newline is refused. A forged leading marker is
also refused. The numeric fingerprint is a compact observation, not the trust
gate; exact source regeneration is the gate.

## Real carrier execution

`presence/concept-pl-10000-13-live.fk` asks Form itself to write each generated
source under `/tmp/coherence-cp10-pl`, recover it, run the target carrier, and
compare target stdout with Form's semantic evaluator. The four examples span an
unmatched pronoun (`you`, ID 0), a mapped action (`go`, ID 46), a concrete noun
(`water`, ID 377), and a control/trajectory word (`loop`, ID 6196).

Nine observed carriers each ran all four programs:

```text
JavaScript  TypeScript  C++  C#  Go  Rust  Ruby  PHP  Swift
```

Result: 9 languages, 36 available, 36 written, 36 recovered, 36 executed, and
36 exact outputs. Adjacent concepts changed output in all 27 comparisons. The
asserted live-execution band returned `1023`.

Python and C were not executed, by explicit task policy. Their program surfaces
are still present in the 130,000-source generation/recovery audit. Java and
Kotlin surfaces are also audited, while their absent local runtimes remain
honestly unexecuted. C++ is the separately named C++ carrier already observed
in the preceding lane; no C program was compiled or run.

Representative JavaScript target outputs showed the materially different
behavior shape:

```text
you   below-unmatched:265:0
go    below-exact:504:5
water below-exact:1133:4
loop  holds-exact:1110:5
```

The generated `water` surface carried real conditioned constants:

```text

```

## Independent proof

The pure band perturbs gloss and relation bytes, then independently perturbs
mapping method, synset, POS, lemma, sense count, POS count, tag count, relation
count, and relation bytes. It also checks all thirteen language generators,
marker parsing, absence of a canned result, and materially changed evaluation.

```text
fkwu        65535
Go          65535
Rust        65535
TypeScript  65535
```

No Python was used. No C source was executed or changed. The temporary C seed
did not grow.

## Honest edge

This closes the previous one-concept PL coverage gap: every anchor now has an
addressed, executable, recoverable surface in each lens. It does not yet mean
that arbitrary natural-language intent can be synthesized into arbitrary
programs, nor that the 2,629 unmatched lexical anchors gained WordNet meaning.
Those remain separate build orders.

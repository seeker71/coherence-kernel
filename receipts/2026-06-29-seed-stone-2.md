# Receipt — seed stone 2: recursive Form recipes run from SOURCE on Windows (defn + arg + call) (2026-06-29)

**Stone 2 of the bootstrap seed.** The source parser (`fk_sparse`) now handles **`defn`** (single arg), **bare
symbol → arg** (tag 2), and **self-call** `(f x)` (tag 7) — so a recursive recipe runs straight from a `.fk` file.
The `defn`'s body becomes `fn[0]`; the arg arrives via `argv[3]`.

## Witnessed native on Windows 11 (`fkwu --src <file.fk> <arg>`)

```
(defn f (n) (if (le n 0) 0 (add n (f (sub n 1)))))   f(5)  -> 15    f(60) -> 1830
(defn fac (n) (if (le n 1) 1 (mul n (fac (sub n 1))))) f(5) -> 120   f(6)  -> 720
```

Recursion, the arg, the conditional, the arithmetic — all parsed from text and run through the unchanged
`fk_walk`. Stone-1 expressions still work; the numeric-table path and sensors are unregressed.

## The honest wall (named, the same one twice)

`f(70)` returns empty — **the tree-walk C-stack depth wall** (`fk_walk` is one huge function, ~17 KB/frame, so
recursion depth is stack-bound at ~60 on a 1 MB stack). This is exactly the limit the pixel-walk receipt pinned;
it is NOT a parser bug (the recipe is correct — `f(60)=1830` is right). It is the same argument for **lowering**:
a deep recipe's home is `form-asm` → a native loop, not the tree-walker. Stone 2 proves the *grammar* runs from
source; depth is the walker's known bound.

## What landed

`runtime/fkwu-uni.c`: `fk_sparse` gains `defn` / arg-ref / self-call; `fk_run_src(path, arg)` takes the staged
arg (`fkwu --src file.fk <arg>`). Helpers `fk_sym_eq2` (srctext↔srctext) + `fk_sym_end` + `fk_smknode`. Unknown
heads consume to `)` and yield a `0` literal — an honest no-op for un-stoned grammar, never a crash.

## The staircase — where stone 2 sits

- **Stone 1 (done):** literals + core ops + `if` from source.
- **Stone 2 (this):** `defn` + single arg + self-recursion → recursive recipes from source.
- **Stone 3 (next):** `do` (n-ary, fold to tag 69) + `let` (frame slots, tags 109/110/111) + multi-arg `defn`/call
  (tag 12, frame packing) — the bookkeeping the flattener does for frames.
- **Stone 4:** strings + lists (`cons`/`head`/`tail`, the string pool) — what `form-flatten.fk` itself reads.
- **Then:** the parser covers `form-flatten.fk`'s grammar → it flattens the Form flattener once → **Form
  self-hosts**, and this C bootstrap parser retires (telos met).

Each stone is a real grammar feature, witnessed. The hard mass is stones 3–4 (frames, strings) — essentially the
flattener's lowering. The **Mac's platform-neutral seed** remains the shortcut to the full grammar immediately;
the stones are the toolchain-free climb to the same place.

## Reproduce

```
gcc -O2 -o fkwu.exe runtime/fkwu-uni.c -lws2_32 -lwinmm -lavicap32 -luser32 -lwlanapi -lbthprops
printf '(defn fac (n) (if (le n 1) 1 (mul n (fac (sub n 1)))))\n' > fac.fk
./fkwu.exe --src fac.fk 5    # -> 120
```

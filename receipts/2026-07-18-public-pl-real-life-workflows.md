## What moved

`presence/concept-pl-real-life-workflows-live.fk` turns the six public algorithm
families into named work contracts without replacing their sources with toy
Form examples.  It rereads and analyzes all 78 attributed local Exercism
programs through the existing native scanner, balanced-group AST, and
family-specific semantic IR.  One varied language is then exposed in full for
each concrete workflow:

Seventy-seven carriers are byte-identical to their pinned Git objects. The C#
Luhn carrier has one explicitly recorded trailing POSIX newline normalization;
its upstream and local hashes both remain named in the public-source receipt.

| real work | public family / selected source | input contract | operational contract |

These inputs and results are explicit **work contracts**.  The source programs
are observed structurally and semantically; Form does not compile or execute
the thirteen host languages.  The door reports that boundary directly rather
than converting an analysis witness into a runtime claim.

## Content-derived world evidence

World admission does not read a scenario title, example input, expected result,
or source filename.  It starts only after source admission.  A label must occur
inside an actually satisfied semantic role name or evidence phrase and must
resolve exactly in the pinned top-10k lexical substrate.  The six admitted
semantic observations were:

| source-derived role label | stable concept id | why the IR supports it |
|---|---:|---|
| `account` | 1466 | Luhn account-input, account-length, and account-digit roles |
| `search` | 1105 | bounded-search and ordered search-interval roles |
| `match` | 1142 | latest-opener and mismatch-rejection roles |
| `factor` | 5812 | divisor, repeated-factor, and factor-output roles |
| `stream` | 5119 | symbol-stream traversal role |
| `grid` | 6130 | grid-position and grid-translation roles |

Each resolved id is persisted through `cwm-persist` using the ordinary kernel
world model.  Positions `1..6` are workflow observation slots, explicitly not
physical object positions, depth, or camera pose.  The live result was six
supported semantic concepts and six known-cell world entities.

## Honest boundary

This is real public program content meeting six non-toy operational contracts.
It is not arbitrary program understanding, native execution of the thirteen
host languages, proof that these examples were executed, or a learned code
model.  The gain is narrower and substantive: source bytes, syntax structure,
semantic obligations, counterfactual failures, and content-derived persistent
world identities now meet at one live production door.

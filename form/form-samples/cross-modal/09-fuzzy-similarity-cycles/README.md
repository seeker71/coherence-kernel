## What's walked

1. **Source text** (1395 chars, same as v1): CLAUDE.md "How This Body Is Tended"
2. **Cycle**: `source ‚Üí S1 (summarize) ‚Üí E1 (expand) ‚Üí S2 (summarize) ‚Üí E2 (expand)`
3. **Per-artifact fuzzy feature recipes**: each axis carries memberships
   (Œº √ó 1000) over the schema's vocabulary, not single tokens
4. **Fuzzy Jaccard similarity** with Zadeh standard t-norm (min) /
   t-conorm (max): `Œ£ min(Œº_A, Œº_B) / Œ£ max(Œº_A, Œº_B)`
5. **Five cycle metrics**: src‚ÜîS1, src‚ÜîE1, src‚ÜîE2, S1‚ÜîS2, E1‚ÜîE2

## What this proves

- **Bit-exact `node_eq` was the wrong tool.** Source and E2 share
  ~80% fuzzy similarity but would `node_eq` to 0 at any axis where
  membership distributions differ by even 1.
- **Compression is lossy but bounded.** First compression drops to
  56%, but compression converges to its own attractor (77% S1‚ÜîS2)
  ‚Äî repeated compression doesn't decay unboundedly.
- **Expansion is the recovery engine.** 84% recovery after a single
  expand from a 56% compressed state. Expansion attractor is 94% ‚Äî
  highly stable, much tighter than compression's 77%.
- **Asymmetric dynamics.** Compression has more freedom (many ways
  to summarize 1395 chars to 250); expansion is closer to a function
  (summary ‚Üí expanded form is more constrained).
- **Fuzzy logic is real math.** Zadeh standard operators, fuzzy
  Jaccard, integer membership scaling ‚Äî all implementable in the
  Form kernel three-way. No floats needed; the kernel doesn't
  change; the lattice now carries similarity, not just equality.

## What this doesn't prove

- **Not cross-modal yet.** Cycle is text-only (compress/expand register
  shifts, not modality shifts). Real cross-modal validation runs
  `text ‚Üí music ‚Üí image ‚Üí text` cycles.
- **LLM-as-extractor, not trained model.** Membership distributions
  are my session's reading. A trained multimodal model would be
  deterministic; this experiment is one example.
- **Schema still small.** 5 fixed-vocab axes. Urs's list named more:
  *flow, beats, melody, harmony, spectrum*. Expanding the schema is
  next.
- **Min/max is one t-norm choice.** Product, ≈Åukasiewicz, and Drastic
  are all proper alternatives with different sharpness. The substrate
  could carry any of them.

## The architectural shift this represents

v1/v2 used `node_eq` (bit-exact identity). This experiment shows the
substrate can carry **fuzzy similarity** at integer precision using
the kernel's existing primitives. The discipline:

- Fuzzy sets as substrate-resident recipes (pairs of token-string +
  membership-int)
- Standard t-norms (min) and t-conorms (max) implemented in Form
- Fuzzy Jaccard (or any other set-similarity measure) as a Form recipe
- Result is an integer in 0..1000, kernel-attested three-way
- Per-pattern recipe NodeID = the substrate-side identity of the
  similarity-measurement itself

The kernel doesn't need to know about fuzziness. The Form recipes
encode the operators; the substrate identity-machinery works the same.
**Pattern carries through, with measured tolerance.**

# Converted — substrate Python files rendered through the universal codec lattice

Each `.fk` file here was generated by `scripts/substrate-py-to-fk.sh` from its
companion `api/app/services/substrate/<name>.py` source. The pipeline:

```
.py file  →  convert.fk  →  grammars/python.fk parse  →  Recipe tree
                                                            │
                                                            ▼
                                                       emit.fk dispatch
                                                            │
                                                            ▼
                                              emits/form.fk templates
                                                            │
                                                            ▼
                                                         .fk text
```

The same `bin-go` kernel that the rest of the body uses; the same `emit-engine.fk`
walker that ships every emit target; no per-language pipeline code outside data.

## Current coverage

The Python parser at `grammars/python.fk` is the **file-header subset** — it
recognizes module docstrings, imports, `def`/`class` headers, simple
assignments, and comments. Function bodies and class bodies are emitted as
default-composite text (children joined with space) because their Recipe
shapes haven't yet entered the universal-shapes vocabulary that `emits/form.fk`
templates dispatch on.

This means today's `.fk` output is the **shape of the migration arc**, not yet
the executable replacement. As `grammars/python.fk` grows to cover statements,
expressions, control flow, and class bodies, the same script produces richer
output without changing — the pipeline is data-driven; coverage expands as
rows.

## Why save these now

To make the architectural ability **visible** rather than aspirational. The
files demonstrate end-to-end flow on real substrate sources. They are not the
final form of the substrate kernel; they are proof that the path is open.

## Re-generating

```bash
scripts/substrate-py-to-fk.sh api/app/services/substrate/orm.py \
    > form/form-stdlib/converted/orm.fk
```

## Known limitations

- Deep recursion in some larger substrate files can exceed the kernel's
  Go goroutine stack (encountered on `form_decompile.py`). The kernel
  walker switching to an explicit stack composts this in a subsequent
  breath; it is a runtime constraint, not an architectural one.
- Whitespace and newlines collapse to single spaces inside default-composite
  branches — the Recipe tree carries shape, not formatting, until per-
  category emit templates land for the relevant Python statement shapes.

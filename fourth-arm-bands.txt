# PROOF-WALKER (fkc-emit-universal) this manifest gates four-way, and a
# SELF-JIT (fkc-nat-expr / fkc-emit-jit2) that crystallizes hot pure functions
# (tags 1-7,12) to NATIVE and melts them on cool, champion-challenger re-earned
# (jit-lower 15, full-jit-lower 63, champion-challenger 127). The native target
# is Form -> asm BYTES, not clang: form-asm 31 / form-lower 31 / form-macho 31 (mac) /
# form-pe-coff 16383 + form-pe-coff-arm64 63 (windows x64+arm64 object) / bootstrap-host 4095 /
# form-elf 31 (linux/android) -> recipe-dylib 787349 -> codesign 632490, every stage a four-way
# Form recipe; clang is dropped by form-asm's byte-identity conviction gate, kept only as
# oracle. The GPU forward y=W2·gelu(W1·x+b1)+b2 emits to all three GPU doors from one recipe
# family: jte-mlp-fwd-msl (metal/mac) / fptx-ffn-fwd 8191 (cuda/nvidia) / form-glsl-ffn-fwd 7
# (vulkan/android). One row per band:
#
#   <band-stem> <table-emitter> <expected-verdict>
#
# table-emitter names the flattener door the band's table file comes through:
#   fkc — fkc-table-file (flt-band-sources-fns ...)                       no string pool
#   fks — fks-table-file (flt-band-sources-fns ...) (flt-band-sources-pool ...)   string pool
#
# The source list is fourth-shim.fk first (core mirrors + string stones as
# function rows), then every non-core `; preludes:` module in declared
# order (BML-dialect sources ride the source-compiler), then the band file
# (same-name convention form-stdlib/<stem>.fk when no prelude is declared).
# Provenance: every row is gated four-way by validate.sh itself on every
# suite run — the fourth leg's verdict must equal the three walkers' byte
# for byte; scripts/hati_os_kernel_audit.sh sections 19/22/24/25/26 carry

# file only when its source moves outside the walker's op families.
#
# A band is out of this manifest for exactly ONE honest reason: it uses an op
# FAMILY fkwu does not yet carry (the standing walls below). A *divergence* —
# fkwu has the ops but computes a different answer than Go/Rust/TS — is NEVER a
# reason to leave a band unregistered. That is a correctness bug: one kernel is
# wrong. Root-cause it and resolve to four-way (recipe shape or walker) before
# the work ships; if the fix is a deeper walker-emitter change, the divergence
# is still owned and tracked to that fix, never buried by omission here.

A transformer layer's attention block is to be reconstructed from its weight tensors alone.
These are ALL the tensors the attention half of one layer owns, with their shapes:

    blk.N.attn_norm.weight       [4096]
    blk.N.attn_q_a.weight        [4096, 1024]
    blk.N.attn_q_a_norm.weight   [1024]
    blk.N.attn_q_b.weight        [1024, 32768]
    blk.N.attn_kv.weight         [4096,  512]
    blk.N.attn_kv_a_norm.weight  [512]
    blk.N.attn_sinks.weight      [64]
    blk.N.attn_output_a.weight   [4096, 8192]
    blk.N.attn_output_b.weight   [8192, 4096]

Metadata that is also known:
    n_embd = 4096, n_head = 64, key_length (head_dim) = 512,
    rope.dimension_count = 64, n_head_kv = 1, q_lora_rank = 1024, kv_lora_rank = 512.

Note 64 heads x 512 head_dim = 32768, which is attn_q_b's output width. There is no separate
attn_k or attn_v tensor: attn_kv is the only key/value projection and it is 4096 -> 512.

Write the ordered list of computational steps that turn the layer input x [4096] into the
attention output [4096], one step per line, numbered. For each step say what it does to what.

Be specific about all of the following, because each is decided by the tensor list above:
  * how the query is produced, and what attn_q_a_norm is for
  * what is stored in the KV cache per token, and how K and V relate to each other
  * how many of each head's 512 dimensions RoPE rotates, and which ones
  * what attn_sinks does, and exactly where it enters the softmax
  * how attn_output_a and attn_output_b combine to get back to 4096

Output only the numbered steps.
